## 🔄 **Backend Flow**

### **1. Application Startup**

```python
# api/main.py

@asynccontextmanager
async def lifespan(app: FastAPI):
    """Lifecycle manager"""
    global data_service
    
    # STARTUP
    logger.info("Starting Warranty Analytics API...")
    data_service = DataService()           # 1. Create service
    data_service.load_data()               # 2. Load parquet file
    dependencies.set_data_service(data_service)  # 3. Set global instance
    logger.info(f"Loaded {len(data_service.get_data())} records")
    
    yield  # App runs here
    
    # SHUTDOWN
    logger.info("Shutting down...")
```

**Startup Sequence**:
1. FastAPI app initializes
2. `DataService` singleton created
3. Loads `warranty_with_predictions.parquet`
4. Prepares data (adds derived fields)
5. Caches in memory
6. Registers routes
7. Starts listening on port 8000

## 🤖 **ML Pipeline Flow**

### **Complete Training Pipeline**

```python
# train_model.py → ml/pipeline.py

def run_full_pipeline(self):
    # 1. LOAD DATA
    df = self.load_data()
    # - Reads warranty_dump.xlsx
    # - Maps Excel columns to standard names
    
    # 2. PREPROCESS TEXT
    df = self.preprocess_text(df)
    # - Combines description fields
    # - Creates 'combined_text' column
    # - Adds 'assigned_flag'
    
    # 3. VECTORIZE TEXT (TF-IDF)
    features = self.vectorize_text(df['combined_text'])
    # - TF-IDF with max 1000 features
    # - Removes stop words
    # - Creates numerical representation
    
    # 4. TRAIN CLUSTERING (K-means)
    cluster_labels = self.train_clustering(features)
    # - K-means with 50 clusters
    # - Groups similar issues
    # - Returns cluster IDs (0-49)
    
    # 5. GENERATE CLUSTER LABELS
    df = self.generate_cluster_labels(df, cluster_labels)
    # - Extracts top keywords per cluster
    # - Creates human-readable labels
    # - Adds 'rca_cluster_label' column
    
    # 6. RULE-BASED CATEGORIZATION
    df = self.apply_rule_based_categorization(df)
    # - Keyword matching on combined_text
    # - 10 predefined categories
    # - Adds 'category_rule_based' column
    
    # 7. ENHANCED NLP CATEGORIZATION
    df = self.enhanced_categorizer.categorize_dataframe(df)
    # - Extracts negative patterns
    # - Identifies action verbs
    # - Detects components
    # - Maps ECU to system
    # - Adds: issue_type_enhanced, system_area, 
    #        affected_component, problem_type
    
    # 8. ADVANCED NLP (spaCy + Sentiment)
    if self.use_advanced_nlp:
        df = self.advanced_nlp_categorizer.categorize_dataframe(df)
        # - Named Entity Recognition (NER)
        # - Subject-Verb-Object extraction
        # - Sentiment analysis
        # - Severity scoring
        # - Adds: action_type_nlp, severity_score,
        #        severity_level, sentiment_polarity,
        #        issue_summary_nlp
    
    # 9. ADD DERIVED FIELDS
    df = self.add_derived_fields(df)
    # - normalized_status (Open/Closed)
    # - assignment_status (Assigned/Unassigned)
    # - resolution_status (Resolved/Unresolved)
    
    # 10. SAVE MODELS
    self.save_models()
    # - Saves TF-IDF vectorizer
    # - Saves K-means model
    # - Saves scaler
    
    # 11. SAVE PROCESSED DATA
    df.to_parquet('data/processed/warranty_with_predictions.parquet')
    
    return df
```

### **2. Analytics Calculation (useMemo)**

```javascript
const analytics = useMemo(() => {
  if (!data.length) return null;
  
  // CALCULATE DISTRIBUTIONS
  
  // 1. Assignment Distribution
  const assignmentDist = data.reduce((acc, item) => {
    const status = item.assignment_status || 'Unknown';
    acc[status] = (acc[status] || 0) + 1;
    return acc;
  }, {});
  
  // 2. Status Distribution
  const statusDist = data.reduce((acc, item) => {
    const status = item.issue_color_status || 'Unknown';
    acc[status] = (acc[status] || 0) + 1;
    return acc;
  }, {});
  
  // 3. Normalized Status (Open/Closed)
  const normalizedStatusDist = data.reduce((acc, item) => {
    const status = item.normalized_status || 
      (['Dark Green', 'Cancelled'].includes(item.issue_color_status) ? 'Closed' : 'Open');
    acc[status] = (acc[status] || 0) + 1;
    return acc;
  }, {});
  
  // 4. Top 10 Clusters
  const clusterDist = data.reduce(...);
  const topClusters = Object.entries(clusterDist)
    .sort((a, b) => b[1] - a[1])
    .slice(0, 10);
  
  // 5. Top 10 Vehicle Models (handles semicolon-separated)
  const modelDist = data.reduce((acc, item) => {
    const modelStr = item.affected_vehicleproject_model || 'Unknown';
    const models = modelStr.includes(';') 
      ? modelStr.split(';').map(m => m.trim()) 
      : [modelStr];
    models.forEach(model => {
      acc[model] = (acc[model] || 0) + 1;
    });
    return acc;
  }, {});
  
  // 6. Top 10 ECUs
  const ecuDist = data.reduce(...);
  
  // 7. Category Distribution
  const categoryDist = data.reduce(...);
  
  // 8. Enhanced Issue Types (Top 15)
  const enhancedTypeDist = data.reduce(...);
  
  // 9. System Area Distribution
  const systemAreaDist = data.reduce(...);
  
  // 10. Affected Component (Top 10)

  // 12. Model Year Distribution

  return {
    total: data.length,

    unassigned: assignmentDist['Unassigned'] || 0,
    open: normalizedStatusDist['Open'] || 0,
```

### **4. Drill-Down Flow**

```javascript
const handleChartClick = (field, value) => {
  // 1. Filter data
  let filtered;
  
  if (field === 'affected_vehicleproject_model') {
    // Handle semicolon-separated models
    filtered = data.filter(item => {
      const models = item.affected_vehicleproject_model.split(';').map(m => m.trim());
      return models.includes(value);
    });
  } else {
    // Standard filtering
    filtered = data.filter(item => item[field] === value);
  }
  
  // 2. Extract unique filter options
  const filterOptions = {
    normalizedStatus: [...new Set(filtered.map(...))],
    status: [...new Set(filtered.map(...))],
    category: [...new Set(filtered.map(...))],
    enhancedType: [...new Set(filtered.map(...))],
    systemArea: [...new Set(filtered.map(...))],
    component: [...new Set(filtered.map(...))],
    problemType: [...new Set(filtered.map(...))],
    model: [...new Set(filtered.map(...))],
    year: [...new Set(filtered.map(...))],
    ecu: [...new Set(filtered.map(...))]
  };
  
  // 3. Set modal data
  setDrillDownData({
    title: `${value} - Detailed View`,
    field,
    value,
    allRecords: filtered,
    count: filtered.length,
    filterOptions
  });
  
  // 4. Show modal
  setShowModal(true);
};
```

## 📈 **Performance Characteristics**

### **Backend**
- **Startup Time**: ~2-3 seconds (load 12,615 records)
- **Memory Usage**: ~100-200 MB (in-memory DataFrame)
- **API Response Time**: <50ms (data already in memory)
- **Concurrent Requests**: Handles 100+ simultaneous requests

### **Frontend**
- **Initial Load**: ~2 seconds (fetch + render)
- **Analytics Calculation**: ~100-200ms (useMemo)
- **Chart Rendering**: ~50-100ms per chart
- **Drill-Down**: <50ms (client-side filtering)
- **Modal Filtering**: <20ms (array operations)

### **ML Training**
- **Without Advanced NLP**: ~30 seconds
- **With Advanced NLP**: ~5-10 minutes
  - spaCy NER: ~2-3 minutes
  - Dependency parsing: ~2-3 minutes
  - Sentiment analysis: ~1-2 minutes

## 🎯 **Key Design Patterns**

### **Backend**
1. **Singleton Pattern**: DataService (single instance)
2. **Dependency Injection**: FastAPI Depends()
3. **Lifespan Management**: Startup/shutdown hooks
4. **Caching**: In-memory data + stats cache
5. **Error Handling**: Global exception handler

### **Frontend**
1. **State Management**: React useState
2. **Memoization**: useMemo for expensive calculations
3. **Event Handling**: onClick drill-down
4. **Conditional Rendering**: Loading/error states
5. **Component Composition**: Modular chart components

### **ML Pipeline**
1. **Pipeline Pattern**: Sequential processing steps
2. **Strategy Pattern**: Pluggable categorizers
3. **Factory Pattern**: Model creation/loading
4. **Template Method**: run_full_pipeline()
